# Scope

## In
- 単一フェスのタイムテーブル閲覧
- アーティストお気に入り選択
- 時間衝突を見ながらの周り順作成
- 共有URLによるプラン共有

## Out
- 複数年・複数フェス横断の履歴管理
- コメント / SNS 機能
- 高度なレコメンド
- 公式データ連携の自動化
- プレイリスト自動生成

## MVP milestones
1. フェス1件分のデータモデル整理
2. タイムテーブル表示 UI を作る
3. 見たいアーティスト選択と周り順作成
4. 共有URLで閲覧できる状態にする

## Risks
- ステージ間移動時間の扱いを雑にするとプランが実用的でなくなる
- データ取得元によっては更新コストや権利面の確認が必要
- 初期から機能を盛りすぎると MVP がぼやける
