# Scope

## In
- 単一フェスのタイムテーブル閲覧
- アーティストお気に入り選択
- 時間衝突を見ながらの周り順作成
- 共有URLによるプラン共有
- 管理サイドによるイベント / タイムテーブル手動入力
- チケットサイトURLからのイベント情報補助取得
- ユーザーによるイベント登録・修正提案
- canonical event による重複防止
- グループ内の予定比較

## Out
- 複数年・複数フェス横断の履歴管理
- コメント / SNS 機能
- 高度なレコメンド
- 完全自動の公式データ連携
- プレイリスト自動生成
- ログイン必須化
- GPS / 位置情報共有

## MVP milestones
1. canonical event / timetable のデータモデル整理
2. 管理サイドの手動入力画面を作る
3. タイムテーブル表示 UI を作る
4. 見たいアーティスト選択と周り順作成
5. 共有URLで閲覧できる状態にする
6. グループ比較を作る
7. ユーザー登録・修正提案と管理承認を作る
8. チケットサイトURL補助取得を入れる

## Risks
- ステージ間移動時間の扱いを雑にするとプランが実用的でなくなる
- データ取得元によっては更新コストや権利面の確認が必要
- 初期から機能を盛りすぎると MVP がぼやける
- ユーザー投稿を即反映にすると canonical データの品質が落ちる
- チケットサイトごとのHTML差分で補助取得の保守コストが出る
